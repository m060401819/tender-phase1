# Product Phase-2 Plan（产品化改造计划）

## 1. 背景

Phase-1 已实现“工程师可用”的采集、入库、查询与管理能力。
Phase-2 目标是在不破坏现有 API 与管理路径的前提下，升级为“业务人员可直接使用”的网页系统。

## 2. Phase-2 产品目标

1. 提供清晰的来源网站运营列表（状态、周期、上次抓取、上次新增、手动抓取）。
2. 提供业务友好的首页总览（新增提醒、关键入口）。
3. 支持抓取周期产品化配置（1/2/3/7 天）。
4. 提供业务表格化的招标信息汇总清单。
5. 支持导出 Excel 便于线下分析。

不在范围：
- AI 匹配
- 企业资质库
- 投标文件生成

## 3. 实施拆分（建议）

### Step-1（当前）产品化首页与来源列表重构

- 新增 `/admin/home`（复用/增强 dashboard 数据）
- 新增 `/admin/source-sites` 产品化来源列表页
- 来源列表展示：
  - 来源名称
  - 官网链接
  - 启用状态
  - 抓取周期
  - 上次抓取时间
  - 上次新增条数
  - 手动抓取按钮
- README 增加访问说明
- 补最小页面测试

### Step-2 周期配置产品化 + 轻量调度

- 把来源抓取周期标准化为 1/2/3/7 天可选
- 保存到现有 `source_site` 配置字段
- 引入 APScheduler（或等价轻量方案）
- 实现 day/interval 级调度（单实例最小可用）

### Step-3 新增提醒与运营提示

- 来源列表新增“新增 X 条”视觉强化
- 首页新增“最近 24 小时新增”
- 顶部提示区域支持醒目展示

### Step-4 业务化招标汇总清单

- 新增业务导向列表页（滚动浏览）
- 筛选：来源/地区/类型/关键词
- 支持 Excel 导出（openpyxl）

## 4. 兼容性与约束

- 不修改既有 API 路径语义
- 不破坏既有 `/admin/*` 页面访问路径
- 优先复用现有模型与服务
- 前端继续使用 FastAPI + Jinja2（不引入复杂前端框架）

## 5. Phase-3 收口补充（已落地）

1. 一键启动闭环：
- `./scripts/dev_up.sh` 启动后即可直接访问产品页
- app 容器启动链路自动执行：等待 Postgres -> `alembic upgrade head` -> 启动 uvicorn

2. 样板源全量翻页：
- `anhui_ggzy_zfcg` 默认 `max_pages=50`
- 在可配置上限内持续翻页，直到无下一页或触发安全停止条件

3. 自动去重与版本追踪：
- 新增 `dedup_key` 字段与索引（`tender_notice/notice_version`）
- 工作台默认按去重总展示，支持查看版本/重复项

4. 多来源扩展占位：
- 新增来源配置与占位：`ccgp_gov_cn`、`ggzy_gov_cn`、`ccgp_hubei`、`ccgp_jiangsu`
- 来源管理页可见、可编辑、可启停
